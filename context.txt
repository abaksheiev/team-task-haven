1️⃣ Основные сущности (Domain)

User — пользователь системы

Team — команда

Board — доска задач (аналог Trello board)

List — список внутри доски (например, ToDo, In Progress, Done)

Task / Card — отдельная задача / карточка

Comment — комментарий к задаче

Attachment — файлы, связанные с задачей

Activity / Event — для real-time уведомлений


3️⃣ Связи между сущностями
Сущность	Связь
User	может принадлежать многим Teams, может быть назначен на многие Tasks
Team	имеет много Users, много Boards
Board	имеет много Lists, принадлежит Team
List	имеет много Tasks, принадлежит Board
Task	имеет много Comments и Attachments, много Assignees (Users)
Comment	принадлежит Task, оставлен User
Activity	привязан к Task, создаётся User

4️⃣ Реальные сценарии

Создание задачи → создаётся Task, добавляется в List, генерируется Activity → отправляется через WebSocket членам команды

Перемещение задачи между списками → обновляется Task.listId, создаётся Activity

Добавление комментария → создаётся Comment, генерируется Activity для real-time оповещения

Отслеживание просроченных задач → метод isOverdue() в Task

5️⃣ Примечания по архитектуре

Все эти классы — Domain Layer

Repository будет работать с базой, возвращая экземпляры этих классов

Service — реализует бизнес-логику (например, фильтр по критическим задачам, уведомления, статистика)

Controller / API Layer — отвечает за HTTP/GraphQL запросы и WebSocket события
